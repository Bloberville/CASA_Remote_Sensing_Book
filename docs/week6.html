5.1 Weekly Learning Diary

Summary

This week introduced Google Earth Engine (GEE) as a cloud-based platform for processing and analysing remote sensing data, representing a shift away from traditional desktop-based workflows. Rather than downloading and manually processing large datasets, GEE enables analysis to be performed directly on remote servers, allowing access to extensive satellite archives such as Landsat and Sentinel without the need for local storage or significant computational resources.

The lectures highlighted how this approach enables more efficient and scalable analysis, particularly when working with large spatial extents or time-series data. By using a scripting-based environment, workflows can be automated and applied consistently across datasets, reducing the need for repetitive manual processing. This represents a significant advantage over traditional methods, where data handling and processing constraints can limit the scope of analysis.

The practical demonstrated these capabilities through working with satellite imagery in GEE, including filtering datasets, applying basic processing steps, and visualising outputs. This reinforced how quickly large volumes of data can be accessed and analysed compared to desktop-based approaches.

However, several limitations were also evident. The use of a scripting environment introduces a learning curve, particularly for those less familiar with coding, and requires a different way of thinking compared to graphical interfaces. In addition, working within a cloud-based system can reduce visibility over underlying processes, making it important to understand how data is being handled and processed. While GEE is currently free to use, there are increasing indications that usage limits and potential costs may be introduced, which could affect its accessibility in the future. This highlights the importance of considering long-term sustainability and dependency when adopting cloud-based platforms for remote sensing analysis.

Applications

The application of platforms such as Google Earth Engine lies in their ability to enable large-scale and efficient analysis of remote sensing data, particularly where traditional desktop-based approaches would be too slow or resource-intensive. By removing the need to download and locally process large datasets, GEE allows users to work directly with extensive satellite archives, making it possible to analyse changes over large areas and long time periods in a relatively short amount of time.

One key application of this is in environmental monitoring, where GEE has been used to analyse global-scale datasets such as deforestation and land cover change. For example, large-scale forest monitoring projects use satellite imagery processed in cloud environments to detect changes over time, enabling near real-time identification of deforestation events. This would be difficult to achieve using traditional workflows due to the volume of data involved.

GEE is also widely used in urban and climate-related analysis, where time-series data can be used to assess trends such as urban expansion or changes in land surface temperature. The ability to rapidly filter, process, and visualise data allows for more dynamic analysis compared to static desktop-based approaches, particularly when working with multiple datasets over time.

In addition to analysis, GEE also enables the development of interactive applications that allow users to engage directly with processed data. These applications can provide a front-end interface where non-technical users can explore outputs, adjust parameters, and visualise results without needing to interact with the underlying code. This extends the use of remote sensing beyond analysts, making it more accessible to decision-makers and stakeholders.

A key advantage of these applications is the ability to develop repeatable and scalable workflows. Once a process has been defined, it can be applied across different locations or time periods with minimal additional effort, supporting more efficient and consistent analysis. This highlights how platforms such as GEE enable remote sensing to move beyond small-scale, manual analysis and become a more integrated and scalable tool within wider analytical workflows.

Reflection

This week highlighted a clear shift towards more efficient and scalable approaches to remote sensing analysis through the use of Google Earth Engine. Compared to previous tools such as SNAP, I found GEE significantly more accessible and intuitive, particularly due to its scripting-based environment. Having a prior understanding of JavaScript made it easier to engage with the platform, and the ability to write and modify code directly provided a greater sense of control over the analysis process.

What stood out most was the speed and efficiency with which large datasets could be accessed and processed. Tasks that would typically require downloading and managing large volumes of data could instead be completed within a single script, highlighting the advantages of cloud-based workflows. However, this also raised some limitations. While scripting provides flexibility, it requires a clear understanding of the syntax and structure, and I found that AI-based tools such as large language models were not always reliable when generating GEE-specific code. This reinforces the importance of understanding the underlying logic rather than relying entirely on automated assistance.

From a practical perspective, I can see clear opportunities to apply this approach within my own work, particularly for analysing change over time or working with larger datasets more efficiently. The ability to develop repeatable workflows and quickly generate outputs would provide a significant improvement over more manual methods. However, it is also important to recognise the constraints around data security. As a cloud-based platform, GEE is not suitable for handling sensitive or restricted data, meaning that its use would need to be carefully considered within organisational policies and data governance frameworks.

Looking ahead, this week has reinforced the value of moving towards more automated and scalable approaches to remote sensing. While traditional tools still have their place, platforms such as GEE offer a more efficient way to work with large and complex datasets. Overall, this has influenced how I approach remote sensing, shifting the focus towards workflows that prioritise efficiency, reproducibility, and integration into wider analytical processes.
